## Mapping Spoken Language to Deterministic Execution Sequences: A Seq2Seq Approach to Real-Time Voice User Interfaces

While prior work in Voice User Interfaces (VUIs) has largely relied on rigid keyword matching or generalized large language model (LLM) API calls, these approaches often suffer from high latency, nondeterministic outputs, and limited robustness to live speech transcription errors. This paper introduces a novel, low-latency end-to-end framework that maps unstructured, live-transcribed speech directly into deterministic macro execution sequences for real-time software control.

Our framework couples a live OpenAI Whisper transcription stream with a domain-specific sequence-to-sequence (Seq2Seq) model trained to translate natural language commands into structured action sequences. Unlike intent classification models that predict a single action label, the Seq2Seq architecture generates ordered command sequences capable of representing multi-step user instructions, enabling more expressive and flexible interaction with interactive applications such as the NES emulator environment.

The primary innovation of our work is a correction-aware execution memory layer that operates downstream of the Seq2Seq model. This layer maintains a temporal representation of generated command sequences, allowing the system to reconcile delayed transcript revisions produced by Whisper before execution. By resolving edge cases where speech recognition corrections occur after a recording window has closed, our architecture preserves deterministic application behavior while maintaining low-latency, real-time responsiveness. Together, the Seq2Seq translation model and execution memory layer provide a robust framework for mapping conversational speech into reliable software execution sequences.
